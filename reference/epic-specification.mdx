## What this reference covers

The EPIC core specification defines:

- the required files and overall file layout
- which files are authoritative when facts conflict
- what agents may write during normal operation
- how `SKILL.md` and `EPIC.md` divide responsibility
- how an agent should resume work on an epic
- the portable state files under `runtime/`
- validation, compatibility, and portability rules

## Core rules at a glance

- Every epic MUST contain `SKILL.md` and `EPIC.md`.
- `SKILL.md` is the discovery and activation surface.
- `EPIC.md` is the durable operating definition.
- `runtime/state/core.json` is authoritative over `runtime/state.json` when
  the split-state form exists.
- Agents MUST treat `policy.yml` as a hard constraint when it exists.
- Reserved fields and machine-readable files MUST round-trip without silent
  loss.
